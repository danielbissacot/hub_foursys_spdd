# 🏃‍♂️ Fase 1: Prompts de Histórias

Esta pasta abriga os prompts customizados para **Product Owners**, **Scrum Masters** e **Analistas de Negócio** da equipe.

## 🎯 Objetivo

Estes templates foram desenhados para acelerar a escrita, o detalhamento metódico e a validação de histórias de usuário, transformando pedidos genéricos em épicos/tasks robustas validadas pelos frameworks de produto (INVEST/BDD).

### Como usar os prompts desta seção

Diferentemente dos prompts de engenharia que rodam em cima de código `.java`, estes templates foram criados para analisar anotações, descrições cruas e rascunhos de clientes.

1. Selecione o texto bruto do seu requerimento / anotação de negócio na mesma tela do seu assistente de IA.
2. Copie a base do comando `.md` desejado desta pasta.
3. Peça para a Inteligência Artificial aplicar a regra sobre o seu texto selecionado, agindo como um "Pair Reviewer" (Parceiro de Auditoria Ágil).

## 📄 Catálogo Atual

| Arquivo | Uso Direto |
|---------|---------|
| `refinamento-agil.md` | Avalia o rascunho de uma história com base na matriz INVEST (Aprovada/Reprovada). Se falha, reescreve o fluxo, aponta os riscos de negócio, destaca as regras `Condição → Ação` e deriva os requisitos técnicos diretos para os desenvolvedores. |
