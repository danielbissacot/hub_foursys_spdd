# 🛡️ Fase 3: Prompts Validadores

Esta pasta é o principal canivete-suíço dos **Desenvolvedores de Software**. Contém "Personas de IA" prontas para funcionarem como revisores de arquitetura e qualidade dentro da própria IDE.

## 📄 Catálogo Atual

| Arquivo | O que este prompt faz |
|---------|---------|
| `validacao-hexagonal.md` | Audita o código em busca de violações da Arquitetura Hexagonal (isolamento de Domínio, uso correto de Adapters e Inversão de Portas). |
| `code-review-spring.md` | Revisão severa de qualidade Java, aplicando Guarda de Cláusulas, SOLID, injeção blindada via construtor e compliance transacional (BigDecimal). |
| `code-review-angular.md` | Revisão focada em Frontend, aplicando OnPush, proteção contra memory leaks n RxJS, descarte do tipo `any` e migração visual para Signals. |
| `testes-unitarios-spring.md` | Gera suítes impecáveis de testes de comportamento via JUnit 5, AssertJ e Mockito, com estrutura BDD e cobertura de exceções e anomalias de regra. |
| `testes-unitarios-angular.md` | Avalia classes TypeScript Angular v21+ gerando testes (`.spec.ts`) modernizados com Vitest/Jasmine, cobrindo reatividade com Signals e TestBed. |
| `diagrama-sequencia.md` | Engenharia reversa visual que desenha toda teia de interações e integrações de um Controlador/Serviço através de um gráfico de sequência local pelo Mermaid. |
