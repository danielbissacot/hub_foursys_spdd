Instrução Completa para o Chat
"Olá, Assistente de IA! Por favor, configure este microserviço para usar o AI Governance Hub com estes 3 passos: 

1. No terminal, execute: 
Remove-Item -Path agentes_foursys -Recurse -Force -ErrorAction SilentlyContinue; git clone --branch hub-ia-arquitetura --depth 1 https://github.com/danielbissacot/ai-governance-hub.git agentes_foursys; code agentes_foursys

2. ROBÔ GITHUB: Crie o arquivo .github/workflows/hub-sync.yml na raiz com o conteúdo oficial do Hub:

name: Hub AI Sync
on:
  push:
    branches: [ main, master, develop ]

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Security Check
        run: |
          if [ -d "agentes_foursys" ]; then
            echo "🛑 ERRO! Remova a pasta 'agentes_foursys' do commit."
            exit 1
          fi

      - name: Sync AI Hub
        run: |
          git clone --branch hub-ia-arquitetura --depth 1 https://github.com/danielbissacot/ai-governance-hub.git agentes_foursys

3. Me avise quando terminar para eu usar o #."

💻 Protocolo de Desenvolvimento (IDE)

A eficiência do Hub reside na Injeção de Contexto Estratégico. Siga o protocolo de invocação para garantir a conformidade:

#

Invocação Especialista

Use o # para selecionar o Agente (Persona) que domina a stack do arquivo atual.

⚙️

Alinhamento Ativo

Aguarde a IA solicitar a Skill ou Fase do Playbook para alinhar a estratégia de execução.

💎

Entrega de Valor

O código gerado herda automaticamente as regras de Clean Code e Segurança do Hub.

📜 Conformidade Arquitetural (instructions)

A "Constituição" do Hub define as fronteiras do que é permitido. Ela atua em background durante toda a sessão.

🛡️ AppSec (Security by Design)

Prevenção ativa contra vulnerabilidades (OWASP), vazamento de credenciais e compliance LGPD.

💎 Design Patterns (Best Practices)

Imposição de Clean Code, SOLID e Arquitetura Hexagonal como padrões não-negociáveis.

🛠️ Arsenal de Skills de Elite

Entenda a hierarquia de especialistas do Hub. Cada agente possui seu próprio conjunto de habilidades específicas.

1 Onde moram os Agentes?

Pasta Raiz: catalog/agents_skills/

  • 📁 spring_boot/ ➜ #AGENTE_SPRING_FOURSYS.md
  • 📁 angular/ ➜ #AGENTE_ANGULAR_FOURSYS.md
  • 📁 cobol/ ➜ #AGENTE_COBOL_FOURSYS.md
2 Onde estão as Skills?

Pasta Interna: .../[tecnologia]/skills/

As Skills são as ferramentas do agente. Elas estão dentro da subpasta de cada tecnologia, prontas para serem invocadas.

🚀 Como usar como um Expert:
#Passo 1: Chame o Especialista (#AGENTE_...)
#Passo 2: Peça a Habilidade desejada ([TESTES], [KAFKA], etc.)
#Passo 3: Siga o roteiro da Fase (#FASE3_...)
"Olá #AGENTE_SPRING_FOURSYS.md, aplique sua skill de [TESTES] seguindo o playbook de #FASE3_VALIDACAO_HEXAGONAL.md."
