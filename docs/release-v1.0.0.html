⚠️ Pré-Requisitos

🗑️
Desinstalar versão anterior
Se já houver uma versão anterior do Foursys SDD Hub instalada no VS Code, ela deve ser desinstalada antes de instalar esta versão.

No VS Code: Extensions → Foursys SDD → Uninstall

Remova também a pasta agentes_foursys/ do projeto caso ainda exista.
💻
VS Code obrigatório
Esta extensão funciona exclusivamente no Visual Studio Code.

Versão mínima requerida: VS Code 1.96.0 ou superior.

Não é compatível com outros editores (IntelliJ, Eclipse, Vim, etc.).
🤖
GitHub Copilot do Banco
É obrigatório estar conectado ao GitHub Copilot licenciado pelo Banco.

O plugin não funciona com contas pessoais ou trials — apenas com a conta corporativa provisionada pela instituição.

Extensão GitHub Copilot deve estar instalada e autenticada no VS Code.

Visão Geral

O Foursys SDD Engine Hybrid é uma extensão VS Code que orquestra o ciclo completo de desenvolvimento orientado a software design (SDD): da especificação à implementação e testes, com governança de IA via GitHub Copilot. Suporta múltiplas stacks, mantém o hub de skills fora do projeto e permite customização de skills pelo desenvolvedor.

🏗️
Multi-stack
Angular, Java/Spring Boot, Node.js, COBOL e Genérica — auto-detectados ou selecionados manualmente.
🤖
Governança de IA
Skills e constituição injetadas no Copilot por stack e design system. Contexto isolado por projeto.
🌐
Hub Global
Um único clone do hub compartilhado entre todos os projetos. Sem poluição de repositório.

Stacks Suportadas

Detectadas automaticamente pelos arquivos do projeto ou selecionadas via botão Trocar Stack.

🅰️ Angular 18+ ☕ Java / Spring Boot 🟢 Node.js 🖥️ COBOL 📦 Genérica

Workflow Dev — Ciclo SDD

Sequência guiada de passos ativados pelos botões da sidebar. Cada botão aciona o participante @foursys_sdd com o contexto correto automaticamente.

0
🏛️ Constitution
Gera os princípios de governança, padrões de código e arquitetura para a stack ativa. Base de todas as gerações seguintes.
1
🔍 Specify
Cria ou analisa a User Story em doc_projeto/user_story.md. Quando mockup presente, usa-o como referência visual.
2
📐 Plan
Gera a especificação técnica detalhada com base na User Story aprovada.
3
📋 Tasks
Quebra o plano em tarefas granulares e rastreáveis.
4
🚀 Implement
Executa a codificação assistida pelo Copilot com todas as skills e padrões da stack injetados.

Hub Global — Fora do Projeto

O hub e as skills são armazenados no globalStorage do VS Code — um único local compartilhado entre todos os projetos. Ao clicar Sync em qualquer projeto, o mesmo hub é reutilizado (ou atualizado com git pull).

# ANTES — hub dentro de cada projeto meu-projeto/ ├── agentes_foursys/ ← clone Git (~MB) ├── .github/skills/ ← dezenas de .md └── doc_projeto/ # DEPOIS — hub único fora de qualquer projeto %APPDATA%\Code\User\globalStorage\foursys\ ├── hub/ ← único clone global ├── skills/ ← skills ativas ├── custom-skills/ ← skills do dev └── copilot-instructions.md meu-projeto/ ├── .github/ ← stub mínimo (2 linhas) └── doc_projeto/ ← produto do fluxo SDD
🔄
Cleanup Automático de Legado
No primeiro Sync após a atualização, o plugin remove automaticamente agentes_foursys/ e .github/skills/ do projeto, migrando para o modelo global.
Novo
📋
.gitignore Automático
O plugin atualiza o .gitignore do projeto adicionando as entradas necessárias para evitar commits acidentais de artefatos do plugin.
