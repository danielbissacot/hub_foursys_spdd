Foursys Personal Copilot Sync

🎯 O que é isso?

O processo SDD da Foursys (constitution, specify, plan, tasks) disponível de forma nativa dentro do próprio GitHub Copilot.

Você chama direto no chat, igual chamaria qualquer recurso do Copilot — sem instalar extensão nenhuma, sem poluir o repositório do projeto onde está trabalhando.

"Uma via paralela ao Hub que já existe — não substitui nada, é uma alternativa pra quem usa VS Code puro, IntelliJ, ou o Copilot CLI no terminal."

⚡ Instalação (1 comando)

Abra um terminal PowerShell (integrado no VS Code, no IntelliJ, ou o PowerShell comum do Windows) e cole o comando abaixo. Leva só alguns segundos.

Comando de Instalação 
irm https://raw.githubusercontent.com/danielbissacot/hub_foursys_spdd/main/foursys-personal-copilot-sync/install.ps1 | iex

Depois de instalar, recarregue a janela do editor pra ele reconhecer as novidades:

  • VS Code: Ctrl+Shift+P → "Developer: Reload Window"
  • IntelliJ: feche e abra o programa de novo

O foursys-sdd-orchestrator sabe em qual dessas fases o projeto está e te leva pra próxima sozinho.

Chamar uma fase do SDD

Cada fase (constitution, specify, plan, tasks) existe pra cada stack. No chat do Copilot, digite / e o nome — ele autocompleta. Exemplos:

Exemplos 
/foursys-specify-angular
/foursys-constitution-spring_boot
/foursys-tasks-cobol

🧭 Deixar o Orquestrador decidir

Selecione foursys-sdd-orchestrator no seletor de Agent e pergunte, por exemplo:

"qual a próxima fase do SDD pra esse projeto?"

Ele detecta a stack sozinho, verifica o que já foi feito, e te leva pra próxima fase.

🧑‍💻 Falar com uma persona

Selecione o Agent do especialista da sua stack:
